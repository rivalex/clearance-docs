# Настройки

Панель Настроек управляет тем, как роли и guard'ы отображаются во всей остальной панели Clearance, а также ролью по умолчанию, назначаемой новым пользователям.

## Что делает панель

- **Метаданные отображения** для каждой роли/guard'а - отображаемое имя, описание и санитизированная SVG-иконка + hex-цвет, показываемые во всей панели.
- **Роль по умолчанию** - роль, автоматически назначаемая новым зарегистрированным пользователям, когда `auto_assign_default_role` равен `true`.
- **Массовое назначение роли по умолчанию** - назначение текущей роли по умолчанию в один клик каждому существующему пользователю, у которого её ещё нет.
- **Показ иконок** - переключает отображение иконок во всей панели.

Каждая предоставленная пользователем SVG-иконка проходит через санитайзер на основе allow-list перед сохранением и снова при рендеринге, так что метаданные отображения можно редактировать, не вводя небезопасную разметку в панель.

## Роль по умолчанию и авто-назначение

```php
// config/clearance.php
'auto_assign_default_role' => true, // requires a default role set in Settings
```

::: tip Установите роль по умолчанию перед включением авто-назначения
`auto_assign_default_role` не имеет эффекта, пока роль по умолчанию не настроена в панели Настроек. Сначала установите роль по умолчанию, затем включите флаг.
:::

Используйте **массовое назначение** в панели в любой момент, чтобы задним числом назначить текущую роль по умолчанию существующим пользователям, у которых её ещё нет - полезно после смены роли по умолчанию или после включения `auto_assign_default_role` в приложении с уже существующими пользователями.

## Контроль доступа

Доступ на чтение требует `clearance-access`. Редактирование метаданных отображения, иконок, цветов или роли по умолчанию требует `clearance-settings-write`:

```php
$user->givePermissionTo('clearance-access');
$user->givePermissionTo('clearance-settings-write');
```

| URI | Имя маршрута | Описание |
|---|---|---|

```blade

```

## База данных

| Таблица | Ключевые столбцы |
|---|---|
| `clr_meta` | `subject_type`, `subject_key` (уникальны вместе) - `display_name`, `description`, `icon_svg`, `color` |
| `clr_settings` | `key` (уникален), `value` |
