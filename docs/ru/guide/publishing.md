# Публикация

Clearance публикует три вида ресурсов через теги `vendor:publish`: конфигурацию, миграции и переводы. Конфигурация и миграции обрабатываются автоматически установщиком (см. [Установку](/ru/guide/installation)); эта страница описывает их публикацию вручную, а также публикацию переводов.

## Конфигурация

```bash
php artisan vendor:publish --tag=clearance-config
```

Публикует `config/clearance.php` в ваше приложение. См. [справочник конфигурации](/ru/guide/configuration) для каждого ключа, его значения по умолчанию и назначения.

## Переводы

```bash
php artisan vendor:publish --tag=clearance-translations
```

Это копирует 9 языков в комплекте (`ar`, `en`, `es`, `fr`, `hi`, `it`, `pt`, `ru`, `zh`) в `lang/vendor/clearance/{locale}/ui.php` в вашем приложении. Чтобы добавить новую локаль или переопределить строки в существующей, добавьте туда каталог локали, используя `en/ui.php` как эталонный набор ключей. Все локали содержат идентичный набор ключей, что проверяется отдельным тестом при каждом изменении.

## Переопределение layout панели

Ключ конфигурации `layout` управляет Blade-layout'ом, используемым полностраничными компонентами:

```php
// config/clearance.php
'layout' => null, // host app's default (config('livewire.layout'))
```

Оставьте `null`, чтобы использовать layout Livewire по умолчанию вашего приложения, либо укажите конкретное представление layout, чтобы отрендерить панель внутри собственной оболочки.

## Миграции

Миграции Clearance публикуются и выполняются автоматически командой `php artisan clearance:install` - установщик также публикует и запускает миграции Spatie первыми, если таблица `roles` ещё не существует. См. [Установку](/ru/guide/installation) для полного пошагового процесса и [предупреждение](/ru/guide/installation#run-the-installer) о том, что нельзя запускать обычный `migrate` на новой установке.
